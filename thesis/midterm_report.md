## 1. 课题介绍

### 1.1 课题背景

随着中国社会老龄化程度的不断加深，老年人健康管理已成为公共卫生领域的重要课题。根据国家统计局发布的数据，截至2023年末，我国60岁及以上老年人口达2.97亿，占总人口的21.1%，老龄化趋势日趋明显[1]。与此同时，高血压、糖尿病、心脏病等慢性病在老年群体中的患病率持续攀升，据中国疾病预防控制中心统计，我国老年高血压患病率约为59.2%，老年糖尿病患病率约为30.2%[2]，慢性病的监测与管理需求极为迫切。

在技术层面，智能手机的普及与移动互联网技术的成熟为移动医疗（mHealth）的快速发展奠定了基础。心电图（ECG）作为心脏疾病筛查的重要手段，其人工智能辅助分析技术近年来取得了显著进展。基于深度学习的12导联ECG自动分类研究已取得突破，公开数据集PTB-XL收录了来自德国的21,837份12导联10秒ECG记录，为算法研究提供了高质量的标准化数据基础[3]。然而，目前大多数移动健康应用仅面向个体用户，缺乏面向老年患者与其家庭照护者协同管理的整合解决方案，且少有将AI辅助ECG分析能力直接嵌入移动端应用的完整平台。

基于上述背景，本课题旨在设计并实现一款面向老年人健康监测的移动医疗平台——CareLink，以解决老年慢性病患者日常健康数据记录分散、家庭照护者难以实时了解患者状态、专业ECG辅助分析门槛高等痛点，实现患者自我管理与家庭照护的有机结合。

### 1.2 研究目标和内容

**研究目标：**

本课题的总体目标是设计并实现一套覆盖移动端、服务端与AI推理端的三层微服务架构健康监测平台，具体包含以下三个核心目标：

（1）构建面向老年用户的跨平台移动健康应用，支持生命体征记录、AI辅助ECG分析、药物提醒及家庭照护者协同管理等功能；

（2）设计并实现基于CNN-CBAM-GRU架构的12导联ECG多标签分类模型，在PTB-XL数据集上实现对五类心电异常（NORM、STTC、MI、CD、HYP）的准确识别；

（3）构建高可用的后端服务系统，实现患者与照护者之间的实时数据共享与异常预警通知机制。

**主要研究内容：**

（1）**需求分析与系统设计：** 面向老年患者与家庭照护者两类角色，梳理功能需求与非功能需求，设计系统总体架构与数据库模式；

（2）**移动端开发：** 基于React Native（Expo框架）开发跨平台移动应用，实现用户认证、生命体征记录、ECG可视化与分析、药物管理、健康趋势分析、照护者管理、认知训练等模块；

（3）**后端服务开发：** 基于Spring Boot 3框架构建RESTful API服务，采用PostgreSQL数据库，实现JWT认证授权、健康数据管理、患者-照护者关系管理、异常检测与报警通知等功能；

（4）**AI推理服务开发：** 基于FastAPI框架与PyTorch深度学习框架，设计并训练CNN-CBAM-GRU模型，搭建ECG推理服务，提供12导联ECG多标签分类API；

（5）**系统集成与测试：** 完成三层系统的集成联调，开展功能测试、性能测试与健壮性测试，验证系统的正确性与可靠性。

### 2.2 总体研究方案

CareLink平台采用三层微服务架构，各层之间通过标准HTTP/RESTful接口进行通信，具体如下：

**（1）移动端（Frontend Layer）**

采用React Native（Expo 54.0.33）框架开发跨平台移动应用，使用TypeScript保证类型安全。前端通过Expo Router实现基于文件系统的路由管理，采用React Context API管理全局状态（身份认证上下文AuthContext、字体大小上下文FontSizeContext）。应用针对老年用户特点，提供三档字体缩放（0.85×/1.0×/1.2×）和触觉反馈（Haptic Feedback）等无障碍特性。

**（2）后端服务（Backend Layer）**

采用Spring Boot 3.5.7（Java 21）构建RESTful API服务，部署于AWS EC2实例。使用PostgreSQL关系型数据库存储结构化健康数据，通过Flyway实现数据库版本化迁移管理。安全机制采用JWT（JSON Web Token）实现无状态认证，结合Spring Security实现基于角色（PATIENT/GUARDIAN）的细粒度访问控制。

**（3）AI推理服务（AI Inference Layer）**

采用FastAPI框架搭建轻量级AI推理服务，部署于HuggingFace Spaces平台。核心模型为自主设计的CNN-CBAM-GRU多标签分类网络，针对12导联ECG的时空联合特征进行建模。模型通过API Key鉴权机制保证接口安全，支持12×5000维度的标准ECG输入，输出五类心电异常的概率向量及综合风险等级。

三层系统的总体交互流程为：移动端采集或模拟生成ECG数据后，经后端服务转发至AI推理服务完成分类推理，推理结果存储至数据库并触发异常预警机制，向患者本人及其绑定的照护者推送健康报警通知。

### 2.3 主要研究成果

目前已取得以下主要研究成果：

#### 2.3.1 AI心电分析模型

本课题自主设计了CNN-CBAM-GRU复合架构模型，用于12导联ECG多标签分类。模型在PTB-XL公开数据集上完成训练，该数据集包含21,837条12导联ECG记录，采样频率500Hz，每条记录时长10秒（5000采样点），涵盖NORM（正常）、STTC（ST/T波改变）、MI（心肌梗死）、CD（传导障碍）、HYP（心室肥厚）五类标签。

模型核心架构由以下四个组件构成：

- **空间特征提取器（CNN+CBAM）：** 采用两级卷积块提取12导联的局部时序特征，每级卷积块包含Conv1D（卷积核大小7）、BatchNorm、ReLU激活及MaxPool下采样，之后接入卷积块注意力模块（CBAM），分别在通道维度和空间维度进行自适应特征加权，输出32维特征图；
- **序列上下文建模器（BiGRU）：** 采用双层双向门控循环单元（Bidirectional GRU）对特征序列进行全局上下文建模，隐藏单元数为64，双向输出拼接后得到128维上下文向量，Dropout率0.5以防止过拟合；
- **幅度特征注入（Amplitude Feature Injection）：** 针对每个导联提取峰峰值、均方根、标准差等幅度统计特征，共36维手工特征，与BiGRU输出向量拼接（总维度164），弥补深度特征对绝对幅度信息的不敏感性；
- **分类头（FC Head）：** 全连接层结构为164→128（ReLU+Dropout 0.5）→5，最终输出五类标签的Sigmoid概率，支持多标签同时预测。

模型经过优化后，在PTB-XL测试集上的五类Macro AUC达到**0.9252（92.52%）**，最难判别的HYP类别AUC为0.8764。与ResNet1D基线模型相比，以ResNet1D 1/13.7的参数规模（0.65M vs 8.88M）达到统计等价的整体性能（t=-0.199，p=0.861），并在最难判别的HYP类别上取得4.06个百分点的优势（详见论文第5章实验部分）。

#### 2.3.2 后端服务系统

后端服务系统已实现全部10个业务模块，包含约31个RESTful API接口，主要功能模块如下：

- **用户认证模块（AuthController）：** 支持用户注册、登录、忘记密码、重置密码、Token刷新、查找用户ID等功能，采用基于RefreshToken的双Token机制（AccessToken有效期24小时，RefreshToken有效期7天）；
- **健康数据模块（UserHealthController）：** 支持血压、血糖等生命体征记录的提交与聚合查询，实现基于阈值（收缩压≥140mmHg、舒张压≥90mmHg、血糖≥200mg/dL）的异常自动检测；
- **ECG分析模块（EcgController）：** 接收移动端提交的12导联ECG数据，调用AI推理服务完成多标签分类，将推理结果存储至数据库并生成健康报警；
- **药物管理模块（MedicationController）：** 支持药物信息的增删改查及服药计划管理；
- **照护者管理模块（GuardianController）：** 实现患者与照护者之间的绑定关系管理，支持双向数据查询；
- **通知报警模块（NotificationController）：** 实现基于健康异常事件（HEALTH_ANOMALY）、疾病趋势（DISEASE_TREND）的多类型报警通知推送，支持按照护者关系进行通知路由；
- **疾病趋势模块（NewsController）：** 集成NewsAPI与KCDC数据源，实现疾病趋势新闻的定期自动采集与推送；
- **认知训练模块（BrainTrainingController）：** 实现老年用户认知功能训练游戏的得分记录与历史追踪。

#### 2.3.3 移动端应用

移动端应用已完成全部功能模块的开发，主要实现成果如下：

- **双角色身份认证系统：** 实现完整的注册、登录、密码找回流程，支持患者（PATIENT）与照护者（GUARDIAN）双角色，通过AsyncStorage实现Token持久化；
- **生命体征监测页面（Vitals）：** 支持血压（收缩压/舒张压）和血糖（空腹/非空腹）的手动记录，提供实时分类诊断（正常/偏高/高血压/低血压等）；
- **ECG分析页面（ECGSimulatorScreen）：** 集成12导联ECG数据生成与可视化，采用SVG渲染技术实时显示12条ECG波形，调用后端API完成AI辅助分析，输出多标签分类结果与综合风险等级；
- **健康趋势分析页面（Insights）：** 支持7天/30天/365天多周期健康趋势展示，基于加权算法（血糖35%+血压35%+ECG30%）计算综合健康评分；
- **药物管理页面（Medication）：** 实现药物列表的增删改查，集成expo-notifications实现基于时间表的本地推送提醒；
- **照护者管理页面（Caregivers）：** 支持多个照护者的绑定管理，提供紧急联系人信息的快速访问；
- **无障碍设计：** 实现三档字体缩放（小/标准/大），全局统一应用ScaledText组件保证字体一致性，提升老年用户使用体验。
